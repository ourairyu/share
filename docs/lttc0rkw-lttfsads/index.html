技术选型

前端

为尽量减少初期阶段的研发成本并降低用户的使用门槛，当采用 Web 作为载体向用户呈现，且移动端与 PC 端是至少两套代码实现（PC 端分用户侧和管理侧）。

根据国内的技术使用情况，并考虑到日后有可能找其他人参与开发时的各方面成本，最好是基于 Vue 进行开发（且不用 TypeScript）。

PC 端管理侧可接入我自研的配置化开发框架 Handie 以提高代码可维护性、可读性及开发效率等。

若之后要做原生客户端，可以考虑 Flutter 或 uni-app。

后端

我虽然没什么实际的线上产品后端开发经验，但还是希望后端采用基于 JS 的技术开发，这样一来：

  • 日后视情况我也能够参与到后端开发中；
  • 若要找其他人来开发，用人成本应该也会更低些；
  • 要是用到 Nuxt.js 之类应该会更容易接入。
