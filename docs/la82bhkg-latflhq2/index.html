场景方案配置

表单是用来收集信息的，一般来说，不应在其中掺杂着与收集信息无甚关联的操作而产生分支流程去分散使用者注意力。

因此，场景方案配置页面的一些设计存在问题和疑问，分别在下图中用红框与蓝框标记：

红框标记的部分是我认为与收集信息无关的操作，这些操作放在场景方案详情中是比较合理的，但在配置表单页就很不合适。

蓝框标记的部分按照我的理解，是带着场景和方案 ID 跳到对应的新建页面后，建完要把创建好的指标评估或性能评估的 ID 带回来填进表单，不知理解是否正确？
