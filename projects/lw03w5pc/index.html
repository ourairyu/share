设计

整体架构大致为：

设备设置

Mac mini

家庭数据中心的核心服务器，将其称之为「iHomeBase」，主要负责：

  1. 数据同步；
  2. 远程开发；
  3. 统一应用服务入口。
访问相关设置

用户相关：

  1. 新建专门的用户组与普通用户；
  2. 新用户登录家庭专用的 Apple ID。

网络相关：

  1. 使用网线进行有线连接网络以保障网络稳定性与速度；
  2. 设置静态 IP 替代动态 IP。

系统待机与启动：

  1. 关闭屏保；
  2. 永不关闭显示器；
  3. 断电重启自动登录新用户。

mini-PC 1

在 Docker 中做以下几类服务：

  1. 软路由；
  2. 信息的获取与聚合；
  3. 资源下载。
  • VMware Fusion 管理 Docker 容器教程 · 云原生实验室

智趣盒子

服务搭建

以下在 Mac mini 上的操作若无特殊说明，都是登录的新建用户；如果操作是通过远程的方式，至少要在「共享」中开启「屏幕共享」。

开发服务

  1. 在 Mac mini 的「共享」中设置「远程登录」；
  2. 配置 MacBook Pro 到 Mac mini 通过 SSH 访问的免登录；
  3. 在 MacBook Pro 上配置 Mac mini SSH 登录别名；
  4. 安装 VS Code 的远程开发插件。

文件存储服务

  1. iCloud 云盘中的特定文件夹设置家庭成员间共享（分操作系统用户与 Apple ID 两个层面）；
  2. 将可被其他设备访问的文件都放到用户「公共」文件夹中指定的某个文件夹下，方便未来一同迁移到外置硬盘中，并在「共享」中设置「文件共享」。

需要注意的是，每个用户的「公共」文件夹的读写权限应为：

  • 自己、新建的用户组可以读写；
  • 其他本机用户只能写（会进入「投件箱」）；
  • 「任何人」用户组无访问权限。

iCloud 缓存服务

在「共享」中设置「内容缓存」，参考 Apple 官方《macOS 使用手册》中内容：

  • 内容缓存介绍
  • macOS 中的内容缓存支持的内容类型
  • 规划和设置内容缓存
  • 在 Mac 上设置内容缓存
  • 在 Mac 上更改内容缓存的“储存空间”选项
